# ai-project-tracker-partyrock
# AI-Driven Project Tracker & Freelancer-Client Communication Hub

An intelligent, transparent project-tracking application built on **AWS PartyRock**. This solution bridges the communication gap between freelancers and clients by converting complex project updates into clear, milestone-driven, and professional status reports.

🔗 **[Live Demo: Try the AWS PartyRock App Here](https://partyrock.aws/u/bismafatima/cbdKoclUR/Project-Tracking-Hub-for-Freelancers)**

---

## 📌 The Problem It Solves
Miscommunication, scope creep, and opaque milestone tracking are leading causes of friction in remote freelance contracts. Non-technical clients often struggle to understand technical updates, while freelancers spend valuable hours drafting status emails instead of working. 

This application automates and standardizes that pipeline, generating:
* Automated milestone breakdowns from raw project scopes.
* Client-ready, professional status updates from rough developer notes.
* Risk mitigation alerts for potential deadline delays.

## 🛠️ Tech Stack & AWS Services Used
* **AWS PartyRock:** Low-code generative AI application development environment.
* **Amazon Bedrock Foundation Models:** Leveraged for advanced natural language processing, text summarization, and prompt-driven logic text generation.
* **Prompt Engineering:** Tailored system instructions designed to prevent hallucinations and maintain a structured, professional business tone.

---

## 🚀 Key Features & App Architecture

The application utilizes a modular widget architecture to process project data sequentially:

1. **Project Scope Ingestion (User Input):** Takes raw parameters such as project goals, tech stack, timelines, and client preferences.
2. **Milestone & Sprint Blueprint Generator (AI Engine):** Breaks down the scope into logical, transparent project phases with clear deliverables.
3. **Rough-to-Professional Update Converter (AI Engine):** Translates casual or deeply technical engineer notes into polished, high-empathy client communications.
4. **Risk & Delay Predictor (AI Engine):** Analyzes current blockers and proactively suggests alternative timelines or mitigation phrasing to preserve the client relationship.

---

## 🧠 Prompt Engineering Philosophy
The core intelligence of this application lies in its prompt structuring. The underlying foundation models are bound by strict system-level constraints to ensure:
* **Tone Control:** Maintaining an empathetic, assertive, and professional corporate tone.
* **Formatting Consistency:** Forcing outputs into clean Markdown tables and bulleted highlights for optimal scannability.
* **Context Preservation:** Ensuring the AI accurately respects deadlines and budget guardrails provided in the initial input.

> The optimized system prompts and logical structure utilized in this app's backend can be viewed in the `/architecture` folder of this repository.
